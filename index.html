  • Easy to implement

Web Feature Service(WFS)


  • The WFS operations support INSERT, UPDATE, DELETE, QUERY and DISCOVERY operations on vector geographic features using HTTP as the distributed computing platform
  • QUERY and DISCOVERY are mandatory
  • The basic interface, like WMS, allows user/client to specify Bounding Box (AOI) and Coordinate Reference System
  • The WFS FILTER specification defines how to use OGC Query Language to perform query operations (same as Catalog)
  • Returns features as GML encoding (default)

WFS service gives the data.


WFS gets operable feature data from multiple servers


Web Coverage Service(WCS)


  • Query and access raw or processed satellite imagery, digital elevation models, raster data, and other types of coverage data
  • Unlike WMS, WCS data can be used as an input to your model

    • OGC allows multiple services to be consume together

    WebGIS Technologies
    -Main Components-

    1. Database Server
    PostgreSQL/PostGIS


    • Database storage and management
    • Enable information to be organized, visualized, and analyzed
    • PostGIS spatially enables PostgreSQL
      • Can store spatial objects
      • Manipulates spatial objects just like other objects in the database
      • helps create complex spatial relationships
      • Spatial querying using SQL
    • Indexing for faster query or map response (GiST, R-Tree etc)

    2. GIS Server : Mapserver


      MapServer is a popular Open Source project whose purpose is to display dynamic spatial maps over the Internet. Some of its major features include:
    • Support for display and querying of hundreds of raster, vector, and database formats
    • On-the-fly projections
    • Support for tiled datasets
    • Progammatic access to Mapscripts APIs (Using Perl, Python, Tcl, PHP, Java)
    • Spatial Indexing for Shapefiles

    2. GIS Server : Geoserver


      GeoServer is an open source server for sharing geospatial data. Designed for interoperability, it publishes data from any major spatial data source using open standards.
    • Outputs  a number of open standards such as Web Feature Service (WFS), Web Map Service (WMS), and Web Coverage Service (WCS)
    • Share and edit geospatial data
    • Open standards allows publishing from many data sources, Eg, Shapefiles, PostGIS SqlServer etc
    • Integrate With Existing Mapping APIs (Google, Yahoo, Bing etc)

    3. Data cataloging : Geonetwork


      GeoNetwork is a catalog application to manage spatially referenced resources.
    • Provides web interface to search and discover geospatial data across multiple catalogs
    • Publish & describe resources
